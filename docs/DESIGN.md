## 零、范围说明

**本地独立 Web 应用**，与飞书无关联，视觉风格沿用飞书对话中心。

| 页面 | 路径 | 说明 |
|------|------|------|
| **对话页面** | `/` | 欢迎区 + 技能快捷入口 + 输入框 + 消息流 |
| **技能广场** | `/skills` | 15 个技能完整展示（分类浏览） |

左侧导航仅 **2 项**：发起新对话、技能广场。

## 一、产品定位

```
用户输入 / 点击技能卡片
         │
         ▼
   猛犸 Backend（透传）
         │
         ▼
      OpenClaw  ──► 自行识别意图、调用对应技能
         │
         ▼
      返回回复 ──► 前端展示
```

**核心原则**：
- 前端**不做**技能路由、**不做**技能与 model 映射
- 技能广场仅是**展示 + 快捷填入示例 prompt**
- 所有对话内容原样送到 OpenClaw，由 OpenClaw 决定用哪个技能

### 技能定义 Schema（`skills/skills.yaml`）

```yaml
skills:
  - id: protein_purify
    name: 蛋白质纯化实验
    category: 实验设计
    icon: "🧬"
    description: 设计完整的蛋白质纯化实验方案
    example: 帮我设计一个 His-tag 蛋白的镍柱亲和纯化流程
```

字段说明：

| 字段 | 用途 |
|------|------|
| `id` | 唯一标识（前端用，不传给 OpenClaw 做路由） |
| `name` | 卡片标题 |
| `category` | 分类 Tab 过滤 |
| `icon` | 卡片图标 |
| `description` | 卡片副标题 |
| `example` | 点击卡片时填入输入框的示例 prompt |

> **不包含** `openclaw_model`、`system_prompt`、`keywords` 等路由字段。

## 五、OpenClaw 对接

### 5.1 调用方式（已实测）

```bash
curl -X POST http://127.0.0.1:18789/v1/chat/completions \
  -H "Authorization: Bearer a5c48bbab43af2819dd167088ca27a84bd6d3ff5ac81304e" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [{"role": "user", "content": "介绍下你自己"}],
    "model": "openclaw"
  }'
```

- 统一使用 `model: "openclaw"`
- 意图识别与技能调度**全部由 OpenClaw 内部完成**

### 5.2 后端接口

#### `POST /api/chat`

```json
// 请求
{
  "session_id": "uuid",
  "messages": [
    {"role": "user", "content": "帮我设计酶进化实验"}
  ]
}

// 响应
{
  "session_id": "uuid",
  "reply": "好的，我来帮你设计...",
  "usage": { "total_tokens": 36831 }
}
```

后端逻辑极简：

```python
# 1. 接收前端 messages
# 2. 原样转发给 OpenClaw（model 固定 "openclaw"）
# 3. 返回 reply
```

#### 其他接口

| 方法 | 路径 | 说明 |

## 八、交互细节

| 操作 | 行为 |
|------|------|
| 点击技能卡片（对话页） | 示例 prompt 填入输入框 |
| 点击技能卡片（技能广场页） | 跳转对话页 + 填入示例 prompt |
| 按 Enter | 发送（Shift+Enter 换行） |
| 发送 | 调用 `/api/chat`，展示 loading → 回复 |
| 发起新对话 | 新 session_id，回到欢迎态（技能广场可见） |
| 切换分类 Tab | 过滤技能卡片 |
| 进入对话态 | 技能广场收起，只显示消息流 + 输入框 |
